I am very much intrigued with reinforced learning. A personal project of making bots in a player-vs-player (playing against other players) and player-vs-environment (usually incorporate min-maxing) settings in video games drove me to write successful algorithms. Some optimizations include, as you might imagine, navigation (optimizing the shortest distance to a moving object) shortest distance around moving objects from A to B (imagine moving spheres, what is the shortest distance to B as the sphere objects move dynamically?), min-maxing against other players (what action would lessen the highest possible success in the opponent?). These are all common problems that Machine learning can be used, and video games are a fantastic way to learn how to code these. Unfortunately, my guide has been taken down, and I no longer have access to it. I wished I backed it up somewhere on an old hard-drive. Some featured I remembed it included were:

  • Rotation prioritization behavior callback looping (an array loop of function callbacks)
  • Overhead reduction (reduce overhead by removing duplicate API calls, ordering API calls based on overhead, applying order of filters based on overhead)
  • Automatic movement: target interception, mesh navigation, positioning navigation (i.e. Navigation from A to B, prediction future location, rogue going behind target quickly and effectively, Line of Sight gain/avoidance)
  • Object dynamic programming and information caching (improves performance substantially using hash maps and arrays of pointer objects)
  • Event driven programming (using combat log event unfiltered to update information, which improves performance)
  • String parsing information (gathering information from descriptions, such as "20% more damage". This information is memoized and is used to determine if an object is "dangerous" or "vulnerable")
  • Augmented API (i.e. computing health including damage reduction and absorption. This allows more powerful information in addition to API calls. These augmented values often needed to be computed)
  • Metamethods and metatables (Lua revolves around tables, so creating powerful metamethods are important for object management)
  • Semi-automatic assistance (ability to override automatic rotation, allow rotation to provide options)
  • Robust settings (allow the user to tailor the bot to his/her liking in very precise settings, over 100 different settings, essentially making no two bots the same)
  • Latency computation (compute values based on latency such as state durations and location, Latency is computed from API call).
  • Human-like behavior (perform action to pass Turing test more effectively, such as clicking a location as a normal distribution so as to not perform on an exact location and providing an option to delay response so it doesn't look like the player responds faster than human reflexes)
  • Forecasting (forecast winning and losing using min-max algorithms and react accordingly, such as running away a losing battle or reacting when 'the stars align')
  • Interproccess communication and Coroutines (Behavior bot and rotation bot needs to communicate in sync. Coroutines can achieve this in Lua as non-preemptive multithreading)
  • Hotkey override and customization (allow the bot to be overridden through hotkeys. Hotkeys can be customized to react various ways, such as 'holding down button', 'perform an action for 1 second after a press'. Some hotkeys, for example, can pauses the bot for a specific duration.)
  • Optimized macros (allow the bot the optimize the user input, such as performing an action only if possible and disabling an action if not. A human would often press a button to perform an action only to be countered before he/she can react)
  • Lua object oriented programming to enforce privacy
  • Weak tables to memoize API calls that are static/dynamic, default values, object attributes
  • ... and many more! (All without any concept of any programming 'standards' or CS degree)
